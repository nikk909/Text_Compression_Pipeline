# run_timestamp 怎么理解

```python
run_timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
```

这行代码的作用是：**取当前时刻，格式化成「年-月-日 时:分:秒」的字符串**。

---

## 拆开看

从左到右三部分：

### 1. `datetime.now()`

- 来自模块 `datetime`（需先 `from datetime import datetime`）
- **现在这一刻**的日期和时间
- 返回一个 `datetime` 对象，例如：`2026-06-23 15:42:08.123456`（还带微秒）

### 2. `.strftime(...)`

- **str** + **f** + **time** = 把 `datetime` **格式化成字符串**
- 括号里是**格式模板**，告诉 Python 怎么显示

### 3. `"%Y-%m-%d %H:%M:%S"` 格式含义

| 符号 | 含义 | 例子 |
|------|------|------|
| `%Y` | 四位年份 | `2026` |
| `%m` | 两位月份（01–12） | `06` |
| `%d` | 两位日期（01–31） | `23` |
| `%H` | 24 小时制小时（00–23） | `15` |
| `%M` | 分钟（00–59） | `42` |
| `%S` | 秒（00–59） | `08` |

中间的 `-`、空格、`:` 会**原样输出**。

---

## 结果

```python
run_timestamp = "2026-06-23 15:42:08"
```

类型是 **`str` 字符串**，适合写进 CSV：

```python
f"{run_timestamp},{iteration_number},..."
```

---

## 和 `time.perf_counter()` 的区别

| | `datetime.now()` | `time.perf_counter()` |
|---|------------------|------------------------|
| 用途 | **记录几点几分**（给人看） | **测代码跑了多少秒** |
| 结果 | `"2026-06-23 15:42:08"` | `1847293.58` 这种差值 |
| 写 results.txt | 用这行 | 不适合当时间戳 |

---

## 一句话

`datetime.now()` 取现在时间，`.strftime("%Y-%m-%d %H:%M:%S")` 转成 `2026-06-23 15:42:08` 这种可读字符串，存进 `run_timestamp`。
