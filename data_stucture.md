各步用什么类型（极简）

| 步骤               | 存什么                                  | Python 类型                                          |
| ------------------ | --------------------------------------- | ---------------------------------------------------- |
| 1 加载             | 很多篇正文                              | `list[str]`                                        |
| 2 预处理           | 每篇变成一串词                          | `list[list[str]]`                                  |
| **3 建索引** | **词 → 哪些文档里有**            | **`dict[str, list[int]]`** ← 后面都围着它转 |
| 5 gap              | 同上键，值改成间隔                      | 还是 `dict[str, list[int]]`                        |
| 6 压缩             | 同上键，值变成字节                      | `dict[str, bytes]`                                 |
| 7 解码             | 先还原成 gap 列表，再还原成 doc id 列表 | 又回到 `dict[str, list[int]]`                      |

所以：**前面是列表装文档；从 Step 3 起核心是字典 + 列表。**

---

## 3 篇文档的最小例子

3 篇文档（`doc_id` 就是 0、1、2）：

```text
doc 0: "cat sat"
doc 1: "cat dog"
doc 2: "dog sat"
```

**Step 2 之后**（每篇一个词列表）：

```python
[
  ["cat", "sat"],   # doc 0
  ["cat", "dog"],   # doc 1
  ["dog", "sat"],   # doc 2
]
```

类型：`list[list[str]]`

---

**Step 3 倒排索引**（核心）：

```python
{
  "cat": [0, 1],
  "dog": [1, 2],
  "sat": [0, 2],
}
```

类型：`dict[str, list[int]]`
含义：键是词，值是 **有序的** 文档编号列表（postings）。

---

**Step 5 对 `"cat": [0, 1]` 做 gap**（列表里还是整数，只是含义变了）：

```python
"cat": [0, 1]   # 0, 1-0=1
```

```python
{
  "cat": [0, 1],
  "dog": [1, 1],   # 1, 2-1=1
  "sat": [0, 2],   # 0, 2-0=2
}
```

类型：仍是 `dict[str, list[int]]`

---

**Step 6 压缩**（每个词对应一段字节）：

```python
{
  "cat": b'\x00\x01',   # 示例，真实字节由你的 VByte 决定
  "dog": b'...',
  "sat": b'...',
}
```

类型：`dict[str, bytes]`

---

## 直接回答你的两个问题

1. **核心数据结构只有一个吗？**

   - **不完全是。** 先有 `list[str]`（文档），再有 `list[list[str]]`（词）。
   - **从建索引开始**，中心就是 **`dict[str, list[int]]`**；压缩时变成 **`dict[str, bytes]`**，编解码再变回带 gap / doc id 的列表。
2. **都是列表吗？**

   - **不是。** 文档用 **列表**，索引用 **字典 + 列表**（每个词一个 posting 列表），压缩后每个词的值是 **`bytes`**，不是列表。

如果你只记一句：**倒排索引 = 字典，键是词，值是该词出现的 doc_id 列表（整数列表，升序）。** 后面 gap、压缩、解压都是改这个字典里的「值」。
